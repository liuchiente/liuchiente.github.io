在下有一點注意力不集中的問題，本來只是想要找個資料，結果不小心看到PHP的Comet問題，又不小心看到一點小demo可能要用到flush的問題，結果就很不小心，開始看PHP的output buffering問題，媽呀，真是不小心。  
  
  
  
先說說為什麼要有output buffering，輸出緩衝很重要嗎，主要是來自於一些比較特殊的狀況，PHP是一種腳本式的語言，當使用者request一個檔案譬如xxx.php時候，想像成讀一個檔案，一個檔案可能會有表頭、表身、表尾，PHP程式內容存在表身，也就因此，當你要使用PHP去輸出一些表頭資料時候，有可能因為檔案本身的表頭已經出去了（jizz），而導致無法再另外送表頭資料這樣。  
  
不過重點也已經不在這邊，也就所以，我們為了避免檔案的表頭已經先Jizz出去，必須使用輸出緩衝區，先讓檔案的輸出來到這邊，避免頭先出去了還想加東西，也就所以啦，使用output buffering的一個好原因就是，讓標頭資訊先Hold on。  
  
所以要使用output buffering ，就要開始啦！下面是相關函數。  
  
ob\_start() 開啟緩衝區  
  
bool ob\_start ([ callback $output\_callback [, int $chunk\_size = 0 [, bool $erase = true ]]] )  
  
ob\_start就是開啟緩衝區，一般來說，可以不加參數也無所謂，他會回傳一個布林值好判斷是否開啟成功，當然也可以丟入相關參數，包含callback函數，緩衝區大小和是否清空緩衝區舊有資料。  
  
ob\_ flush()  
  
ob\_flush不用傳入任何參數，也不回傳值，他的功用主要在於把PHP緩衝區的資料推出，但是推出不代表顯示，更正確的來說，他會把緩衝區資料推出後，然後把緩衝區資料清空，但是這邊來說，他並不會把資料丟給request做一個輸出的動作，所以要做輸出，就要另外使用函數了。  
  
flush()  
  
承上，因為PHP緩衝區的資料被推出來了，但沒有被丟給request做顯示，所以我們緊接的就要用flush()把伺服器上緩衝區的所有資料顯示出來，如此一來我們才能真正的接收到緩衝區的資料，但是需要注意的是，Flush本身是針對伺服器緩衝區做操作，假定今天伺服器平台要做移植，就要考慮是否可用了。  
  
ob\_ clean()  
  
清空PHP緩衝區的資料，但是這清空只是單純把資料清空，並沒做其他動作。  
  
ob\_ end\_ clean  
  
清空緩衝區資料，並且把緩衝區關閉，也就是說你之後不再使用緩衝區，直接關閉了。  
  
ob\_ end\_ flush  
  
把緩衝區資料推出來，並把緩衝區關閉。  
  
ob\_ get\_ clean  
  
取得緩衝區現有的資料，並且把現有的資料從緩衝區砍掉。  
  
ob\_ get\_ contents  
  
取得緩衝區現有資料。  
  
ob\_ get\_ flush  
  
把緩衝區資料以字串形式回傳，並且關閉緩衝區。  
  
ob\_implicit\_flush(INT 0 or 1)  
  
打開或關閉絕對更新，也就是說當我輸出緩衝區，會直接輸出到瀏覽器，就不用再使用flush來把伺服器緩衝區上的資料推到瀏覽器。
